# AIを活用したナレッジ継承・スキルマッチング システム プラン

## 1. システム概要

**目的**: 組織内の暗黙知・経験知をAIで構造化し、適切な人材とスキルをマッチングする

## 2. 主要機能

### A. ナレッジ継承モジュール

| 機能 | 説明 |
|------|------|
| 知識抽出 | ベテラン社員へのAIインタビューで暗黙知を言語化 |
| ドキュメント解析 | 既存資料・議事録・チャットログからナレッジを自動抽出 |
| 知識グラフ構築 | 抽出した知識を関連付けてグラフDB化 |
| Q&Aボット | 蓄積ナレッジに基づく質問応答システム |

### B. スキルセット管理モジュール

| 機能 | 説明 |
|------|------|
| スキル自動検出 | 業務履歴・成果物からスキルを自動推定 |
| スキルマップ可視化 | 組織全体のスキル分布をダッシュボード表示 |
| ギャップ分析 | 必要スキルと現状のギャップを特定 |

### C. スキルマッチングモジュール

| 機能 | 説明 |
|------|------|
| プロジェクトマッチング | 案件要件に最適な人材をレコメンド |
| メンター推薦 | 学びたいスキルに対して適切な指導者を提案 |
| 継承計画自動生成 | 退職予定者の知識を誰に継承すべきか提案 |

### D. AIメンタリングモジュール（新規）

ベテラン社員の思考パターン・判断基準をAIが学習し、24時間対応可能な「分身AI」を提供

| 機能 | 説明 |
|------|------|
| パーソナルAI生成 | 各ベテランの過去回答・資料・メールから個人AIモデルを構築 |
| 思考プロセス再現 | 「なぜその判断？」に対し、根拠と考え方を説明 |
| 状況別アドバイス | 場面に応じた助言（クレーム対応、見積もり判断など） |
| トーン再現 | 本人の話し方・口癖も再現し親しみやすさを維持 |
| エスカレーション | AIで解決できない場合は本人へ自動引継ぎ |

**データソース**: 社内チャット、メール、会議録、1on1記録、インタビュー音声、成果物

**依存関係**: ナレッジ継承モジュール（A）で収集したデータを活用

**倫理・プライバシー配慮**:
- 本人の同意必須
- 生成内容は本人が確認・修正可能
- 「AIの回答です」と明示

## 3. 技術スタック案

```
フロントエンド: React/Next.js
バックエンド: Python (FastAPI)
AI/ML: OpenAI API / Claude API / LangChain
データベース: PostgreSQL + Neo4j（知識グラフ）
ベクトルDB: Pinecone / Weaviate
```
